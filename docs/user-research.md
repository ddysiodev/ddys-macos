# 用户需求与 macOS 功能规划

macOS 端和浏览器扩展的使用场景不同：扩展偏“随手查”，macOS 应用偏“长期打开、集中管理、批量处理”。因此 DDYS macOS 选择做成原生窗口工作台，而不是把扩展弹窗放大。

## 关键需求

- 快速搜索：输入关键词后立即进入结果页，并保留类型筛选。
- 持续发现：长期打开时能刷新最新更新、热门内容和片单。
- 追更提醒：用户关心演员、导演、片名或主题关键词时，需要订阅和通知。
- 本地数据：收藏、稍后看、历史、备注需要跨会话保存，并且默认留在当前 Mac。
- 资源聚合：详情页需要同时看在线资源、下载资源、相关内容。
- 数据可迁移：用户换设备或重装应用时，需要导入导出 JSON。
- 可诊断：API 地址、网络、缓存、返回样本需要一键排查。
- macOS 存在感：菜单栏托盘、通知、WKWebView、`.app` 和 `DMG` 是 macOS 桌面端的核心能力。

## 功能落点

- `发现`：最新、热门、片单。
- `搜索`：关键词、类型筛选、订阅当前关键词。
- `日历`：按月份查看更新。
- `片库`：收藏、稍后看、观看历史。
- `订阅`：添加关键词、暂停、删除、立即检查、通知。
- `设置`：API、站点、主题、缓存、刷新、通知、打开方式、导入导出、诊断。

## 技术选择

- Tauri v2：用系统 WKWebView 打包轻量 macOS 应用，支持 `.app` 和 `DMG` 目标。
- Vanilla JavaScript + Vite：减少前端依赖，便于源码审计和后续迁移。
- 本地优先：核心数据先保存在当前 Mac，避免引入账号和同步复杂度。
- 模块化核心：API、存储、订阅、标准化分层，便于后续复用到其他桌面平台或移动壳。
